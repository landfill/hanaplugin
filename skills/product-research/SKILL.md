# 프로덕트 시장조사 & 기획 프롬프트 빌더

프로덕트 아이디어를 2단계로 발전시킨다:
1. **인터뷰** — 체계적 질문으로 정보 수집
2. **프롬프트 생성** — 수집된 정보로 심층 리서치 프롬프트 생성

## 워크플로우

```
Step 0: 페르소나 선택 (A/B/C)
    ↓
Step 1-8: 페르소나별 인터뷰 (한 번에 한 질문)
    ↓
최종 정리: 수집 정보 요약
    ↓
프롬프트 생성: 맞춤형 리서치 프롬프트 출력
```

<HARD-GATE>
인터뷰를 끝내기 전에 리서치 프롬프트를 생성하지 마라. 모든 단계의 답변을 수집한 후에만 최종 프롬프트를 생성한다.
</HARD-GATE>

## Step 0: 페르소나 선택

AskQuestion 도구로 사용자 배경을 파악한다:

```
질문: "프로덕트 기획을 도와드리겠습니다. 본인에게 가장 가까운 유형을 선택해주세요."
선택지:
  A) 비즈니스/기획자 — 아이디어와 비즈니스 감각은 있지만 기술 배경 없음
  B) PM/프로덕트 오너 — 프로덕트 경험 있고 기술도 어느 정도 이해
  C) 창업가/솔로파운더 — 비즈니스부터 기술까지 모두 알아야 하는 상황
```

## 인터뷰 진행 규칙

### 핵심 원칙
- **한 번에 하나의 단계만** 질문한다 (한 단계에 소주제 여러 개는 괜찮음)
- 답변을 받으면 **간단히 요약**하고 다음 단계 예고
- 답변이 불충분하면 **후속 질문 1-2개**만 추가
- 사용자가 "모르겠다"/"잘 모르겠다"고 하면 **건너뛰고** 다음으로 진행
- 전체 인터뷰는 **8단계**로 구성

### 페르소나별 질문 파일

선택된 페르소나에 따라 해당 질문 파일을 읽고 그 흐름을 따른다:

- **A) 비즈니스/기획자** → [questions-business.md](questions-business.md) 읽기
- **B) PM/프로덕트 오너** → [questions-pm.md](questions-pm.md) 읽기
- **C) 창업가/솔로파운더** → [questions-founder.md](questions-founder.md) 읽기

## 인터뷰 완료 후: 최종 정리

모든 단계가 끝나면 수집된 정보를 아래 포맷으로 요약한다:

```markdown
## 수집된 정보 요약

### 기본 정보
- 프로덕트: [요약]
- 배경: [A/B/C 유형 및 현재 단계]

### 시장 분석
- 시장 상황: [요약]
- 경쟁사: [요약]
- 트렌드: [요약]

### 타겟 고객
- 주 타겟: [요약]
- 페인 포인트: [요약]
- 현재 해결 방법: [요약]

### 프로덕트 전략
- 차별점: [요약]
- 핵심 기능: [요약]
- MVP: [요약]

### 비즈니스 모델
- 수익 모델: [요약]
- 가격: [요약]
- 목표: [요약]

### 실행 계획
- 일정: [요약]
- 리소스: [요약]
- 리스크: [요약]
```

## 리서치 프롬프트 생성

요약을 보여준 후 사용자에게 확인을 받는다:
- "이 요약이 맞나요? 수정할 부분이 있으면 말씀해주세요."

확인 후 [research-prompt-template.md](research-prompt-template.md)를 읽고,
수집된 정보로 `[placeholder]`를 모두 채워서 완성된 리서치 프롬프트를 출력한다.

### 프롬프트 출력 방식

완성된 프롬프트를 마크다운 코드블록으로 감싸서 출력한다.
사용자가 복사해서 다른 AI 플랫폼에 바로 제공할 수 있도록 한다.

### 후속 안내

프롬프트 출력 후 다음 안내를 제공한다:

**추천 AI 플랫폼:**
- **Claude.ai** — 긴 문맥 처리에 강함, 체계적 분석
- **ChatGPT** — Deep Research 기능으로 웹 검색 기반 리서치
- **Gemini** — 대용량 데이터 처리 및 다양한 소스 통합

**사용 팁:**
- 리서치는 15-30분 소요 가능
- 여러 AI를 사용해 교차 검증 추천
- 결과를 받은 후 특정 섹션 심화 가능

**수정 옵션:**
- 특정 섹션 강화/제거
- 추가 분석 항목
- 기타 조정

- [ ] 수집 정보 요약 제시 및 사용자 확인
